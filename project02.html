Advantage of NAT(Network Address Translation)

NAT is a system that converts one public IP to several private IPs. A router with two or more LAN cards acts as an intermediary between the private network and the external network. The router used a lot in the vicinity is a typical NAT system. In addition, the private network is structured so that access from the external network to the internal PC of the private network is impossible unless special settings are made, so the security is strong.

Theory of NAT

When the first sender delivers the packet to the gateway, SRC_IP is filled with the sender's address inside the private network. For communication, the gateway receives the packet and randomly modifies the IP address in the packet to its public IP and sends it to the destination. The recipient who receives the packet will send the packet with the public IP inserted by the gateway in the destination IP address, and the packet will reach the gateway.

VLAN(Virtual Local Area Network)

As a function supported by a switch, it is to create multiple networks in one switch. Multiple networks means that broadcast domains are divided. When networks are divided in this way, communication between each other becomes possible only through routers.

This is similar concept as NAT

Broadcast domains of hosts connected to the same switch can be virtually divided, or broadcast domains of hosts connected to different switches can be combined. Like NAT, when internal vlans go out, the switch converts the vlan into one public address and transmits it. By default, the unused vlan number is put in the distribution switch so that other vlans other than the licensed vlan cannot enter.

Trunking

Originally, one link per VLAN should be allocated, but since too many links are required in that case, trunking is used so that all VLANs can move through one link. At this time, tag each vlan so that it can be properly distributed at the destination.

SVI can be defined as a vlan with IP concept

If SVI communicates with the existing Trunk and Allow different VLANs,this time, the other VLANs communicate with the router. IP communication is performed between the router and the L3 Switch. You cannot use Trunk or Allow. SVI allows communication between VLANs using the router's Routing Protocol. It's a technology that makes it possible.

VTP(vlan trunking protocol)-Protocol created by cisco to manage VLAN

-Server <-> Client concept
-Receives vlan configuration information, but does not receive port information
-Minimization of management resources (set only one and sprinkle with synchronization)
-purpose 2. Minimize database storage resources of vlan configuration information

Condition of VTP

1. Only cisco
2. Must be connected to trunk
3. The vtp domain must be the same (only communication between the same ones)
4. The vtp version must be the same
5. vtp password
6. vtp revision (number of updates)

VTP MODE

Server : Creation, modification, deletion possible, revision number increases each time it is set (information exchange -> synchronization)
If the revision number is the highest, it becomes a server -> If a new switch is not initialized, the network may be damaged.
Client : Creation, modification, deletion impossible, (forward -> synchronization)
Transparent : Can create, edit, and delete (only forward other kids information)

L3 traffic has TTL (Time To Live), so there is no infinite loop.

HSRP(Hot Standby Router Protocol)

When a PC sends a packet to a switch in an environment with two gateways, the switch floods both gates. HSRP operates between the two gateways and manages the MAC table by determining who is active and who is the standby, and when active goes down, the standby manages the MAC table. HSRPv1 and CGMP cannot be set at the same time HSRPv2 and CGMP can be set at the same time

How to manage packets that the L2 switch does not understand and floods?

1. Broadcast : vlan, rate limit
2. Multicast : ICMP snooping -> CGMP(Cisco version of ICMP)
3. Unknown Unicast : This occurs because the mac aging time and arp aging time of the mac table are different -> Align two times, statically make the mac

HSRP as group

If multiple hosts belong to the same HSRP group, a lot of load will occur in the active router. Because everyone is trying to communicate with the Active Router. In this case, if the HSRP group is divided and the active router is set differently, the router that communicates with each group will be different and the load can be reduced.

VRRP(Virtual Router Redundancy Protocol)

Unlike HSRP, which uses an address not set in Interface as Virtual IP, VRRP can set Virtual IP with real IP set in Interface(Of course, you can use a separate Virtual IP)
-Master router selection priority (HSRP active = VRRP master), (HSRP standby = VRRP backup)
1. Router where the configured VIP is the actual interface ip
2. Router with high priority *** (default value = 100)
3. Router with high IP address

In the master router, the vrrp advertisement period is set with the "timers advertise" command

Then this value is notified to the backup router. The backup router learns the vrrp advertisement cycle received from the master router with the "timers learn" command. If the backup router directly sets the cycle with the advertise command rather than the learn command, it becomes the master itself. That is, vrrp is not properly configured.

GLBP(Virtual Router Redundancy Protocol)

GLBP is a CISCO redundancy protocol that enhances the function of HSRP. GLBP, an extension of HSRP, dynamically enforces virtual ip assignments and assigns multiple virtual mac addresses to GLBP group members. GLBP provides load balancing for multiple routers using a single virtual IP address and multiple MAC addresses. The hosts are configured with the same virtual IP address, and packet switching is possible for all routers in the virtual router group at the same time.

The status of a group member is one of active, standby, and listening.

Members of the GLBP group elect AVG and standby virtual gateway (SVG). If there are more than 3 members, the remaining members are in the listening state. When AVG goes down, SVG takes over. Then, among the members in the listening state, SVG is again elected. Even if AVG comes back to life or a new member with high priority appears, AVG and SVG will not change unless preempt is used. AVG allocates virtual mac addresses to each member of the GLBP group. HSRP has one virtual mac in a virtual ip, but GLBP assigns one virtual mac address to each member. However, up to 4 virtual macs can be allocated per group in GLBP.

DHCP(Dynamic Host Configuration Protocol)

There is 4 DHCP message
Discovery : Broadcast message from client to server. Basically, It should be configured in the same vlan. But then it has weakness of security and increased computational load. So assign client and server to different vlans, configure dhcp relay agent, change to unicast in the middle, and deliver to server
agent : L3 function(command: ip helper address), still normal dhcp cannot be controlled(Weak against dhcp starvation attack) -> [Solution 1-port security, 2-user authentication (ieee802.1x-dot1x)]
offer : server's response to discovery (if there are multiple servers) -> dhcp snooping is used to distinguish unauthorized servers (all ports become untrusted-messages from servers coming from all ports are blocked) -> Build in agent -> Directly declare the port where the server message should come in.
request : [1. Client's response to offer / 2. Request message]
ack : Server's response to request

Rental period renewal

When the first renewal is attempted, the renewal is attempted when 50% of the rental period remains. The client directly transmits in the form of a DHCPQUEST message. If the server transmits affirmative ACK, it transmits the updated parameters along with the new IP. When the server is turned off or cannot be received, the client receives a message indicating that it has not been updated. However, since 50% of the rental period is still remaining, the use of TCP/IP is maintained. When 87.5% of the rental period expires, the second renewal is attempted. If the second renewal fails, the lease period expires or when DHCPNACK is received, the DHCP Client stops using the IP address and has to go through the above 4 steps again to request an IP again.

1. dhcp relay agent
2. NAC: port-security. dot1x ...
3. dhcp rate limit
4. dhcp snooping -> binding
5. dhcp option 82 [Authentication between dhcp server and dhcp relay agent] Unauthorized dhcp client using unauthorized ip
6. ip source guard [not assigned ip from dhcp server Unauthorized address communication is not possible]
7. arp inspection: Block when attempting to change the mapping of IP address and mac assigned from dhcp server.
