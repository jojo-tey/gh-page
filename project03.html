Floating Static Routing

Floating Static Routing is a static routing function that automatically configures the backup path in the routing table when the target network path is set to static, but if the configured path goes down.
Static routing has a disadvantage that if the route to the destination network is down, the administrator must detect the change in topology and manually set another route. This is secured by Floating Static Routing.

Path will be changed automatically

When going from S1 to S2, the path is S1->R1->R2->R4->S2.
At this time, when the R1<->R2 path is down, the path is changed to S1->R1->R3->R4->S2 by floating static. Note that if you turn down fa 0/0 of R1, the route to which you are going is changed, but the route to return is not changed, so routing is not possible. The timeout needs to be set in the sla echo packet to recognize network changes.

PBR(Policy Based Routing) with IP SLA(Internet protocol service level agreement)

Cisco IOS IP SLAs continuously collect data about such things as response times, latency, jitter and packet loss. This not only provides the network administrator with baseline information about network performance, it also helps the administrator to verify Quality of Service (QoS) levels and quickly identify the root of a problem if performance levels drop. When integrated with a network management system (NMS), visual alerts of threshold violations can be captured in real-time. IP SLAs are especially useful for wide area networks (WANs) that connect multiple geographies and needs to be monitored from one central location.

RIP(Routing Information Protocol)

The routing information protocol is a routing protocol operating on UDP/IP. RIP is a task of dynamically exchanging routes with neighboring hosts using an algorithm called DVA by quantifying the number of routers that may be routed through the number of hops. Determines the shortest path for a packet to reach its destination network address.

Basic mapping (using ppp: equipment certification, standard)

-Verification of communication with RIP version 2 from R1 to R5 excluding R6
-Currently, there is no customer information and only link information is used to form a rip neighbor (Cisco does not say neighbor in rip)
-Customer advertisement: R5 / R6

EIGRP(Enhanced Interior Gateway Routing Protocol)

It is the distance vector internal gateway protocol (IGP) invented by Cisco. It is used by routers for the purpose of exchanging routing data in autonomous systems.
IGRP was created to partially overcome the limitations of RIP (maximum number of hops 15, one routing metric) when used in large networks. IGRP supports multiple metrics for individual paths, including bandwidth, delay, load, MTU, and reliability.

IGRP is considered a classed routing protocol

Since this protocol has no space for a subnet mask, the router assumes that all subnetwork addresses in the same Class A, Class B, and Class C network have the same subnet mask as the one configured on the interface. This contrasts with classless routing protocols that can use a variable subnet mask (VLSM).

AS12 EIGRP routing / R1<->S1, R2<->S2 RIP

-Neighbor status monitoring (eigrp neighbor update: clear ip eigrp neighbor)
-When the information specified in acl 12 is received from fa0/0 using the offset-list command, the (in) metric is increased by 1.
-Routes that are not optimal disappear from the routing table.
-Using the routing update filtering policy, only the loopback address and default route are transmitted to the server, and the remaining detailed information is blocked.

OSPF(Open Shortest Path First)

OSPF is a routing protocol for Internet Protocol (IP) networks. It uses a link state routing (LSR) algorithm and falls into the group of interior gateway protocols (IGPs), operating within a single autonomous system (AS).
All OSPF areas must be physically or virtually linked to the backbone area (= area0). Because ospf is a link state, it knows its own area information accurately, but for other areas, the ABR router should put summary information. Therefore, the ABR router must be connected to the backbone area.

HUB & SPOKE


Currently, the state of area0 network is performing multicast update and is in full mesh state. I am trying to make the area0 section a hub&spoke structure. In the hub & spoke structure, the DR must be performed by the hub, and the spokes must never do DR/BDR.
When ospf advertises loopback, which is a virtual interface, ospf knows that loopback is not a network because it is a link state. So it advertises as a host. (= 32bit ad)
If you want to advertise in the original bit (16bit), set loopback directly in ospf, or redistribute by specifying the target through the access-list.

OSPF's policy is implemented in border routers

When communicating from R7_lo0 (7.7.7.7) to R1_fa0/0 (1.1.123.1), since R7 to R4 is a single path, it goes through R6 to R4, but in R4 there are two paths. Will you go to R3 or through R5? This decision is determined by LSA-type, not by metric. ospf looks at the LSA-type before looking at the metrics (O> IA> E1> N1> E2> N2)
Path selection procedure [1. PBR 2. longest match rule 3. AD --> LSA-type --> 4. metric]
